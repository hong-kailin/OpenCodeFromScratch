# 5.1 为什么需要持久化

> 本课目标：理解当前 agent 的数据丢失问题，知道为什么要引入数据库。

## 当前的问题

目前我们的 agent 把对话历史存在内存里（`messages` 数组）：

```ts
// 程序启动时，messages 是空的
const messages: Message[] = [
  { role: "system", content: "你是一个助手..." },
]

// 对话过程中，messages 不断增长
messages.push({ role: "user", content: "你好" })
messages.push({ role: "assistant", content: "你好！有什么..." })

// 程序退出 → messages 全部消失！
```

**程序一退出，所有对话历史就没了。** 重启后不记得之前说过什么，每次都从零开始。

## 持久化是什么

**持久化**（persistence）：把内存里的数据存到磁盘，程序重启后还能读回来。

```
内存（RAM）                    磁盘（硬盘）
┌──────────────┐              ┌──────────────┐
│ messages 数组 │  ──写入──→   │ 数据库文件    │
│ 程序退出就没了 │              │ 永久保存      │
└──────────────┘              └──────────────┘
                               │
                    程序重启   │ 读取
                               ▼
                     ┌──────────────┐
                     │ messages 数组 │
                     │ 从数据库恢复  │
                     └──────────────┘
```

## opencode 怎么做的

opencode 把所有对话存到 SQLite 数据库——重启后从数据库加载历史，继续对话。它的数据库文件在 `~/.local/share/opencode/opencode.db`。

> opencode 甚至支持跨进程恢复——你在终端 A 开了对话，关掉后用终端 B 重新打开，能恢复之前的对话。这就是持久化的价值。

## 本课小结

1. **问题**：内存数据程序退出就没了
2. **持久化**：把数据存到磁盘，重启后读回来
3. **解决方案**：用 SQLite 数据库存储对话历史
