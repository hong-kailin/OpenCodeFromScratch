# 🤖 OpenCode From Scratch

> 从零开始，一行一行重新实现 [opencode](https://opencode.ai)--一个开源 AI 编程 agent。

---

## 🤔 为什么有这个项目

### 💡 编码变得廉价，架构变得昂贵

AI 时代，写代码这件事正在变得廉价。一句"帮我写个 XXX"，AI 就能产出能跑的代码。但很快你会发现--

**问题从来不在编码本身，而在怎么设计、怎么组织、怎么验证。**

无脑 vibe coding 的项目，开头几天爽得飞起 🚀：功能飞速增长，屏幕上满是绿色的 diff。但越到后期越力不从心 😵‍💫：

- 改一个地方崩三个地方
- bug 层出不穷，行为不符合预期
- 代码像一团乱麻，自己都看不懂三天前写的什么
- 最后项目变成了一个"能跑但不敢碰"的怪物 🧟

这不是 AI 的问题，是**架构设计**的问题。AI 帮你写了每一块砖 🧱，但如果没有蓝图，砖堆得越多越容易塌。

### 🧭 一个算法工程师的学习弯路

我是一名算法工程师，Python 技术栈，对前后端完全不会。

一开始我就想学 opencode 的源码。opencode 是一个开源的 AI 编程 agent，用 TypeScript + Bun + Effect-TS + Drizzle + opentui 构建，31 个 package 的大型 monorepo。它的架构经过真实用户的检验，是学习"生产级 agent 怎么设计"的绝佳教材 📖。

但直接读源码？太难了 😫。一个文件 import 另一个文件，Effect 服务层层嵌套，Stream 流转来转去，读到第三个文件就迷路了。

于是我退了一步，想"先做几个小项目练练手" 🛠️。感受了 vibe coding 的强大，确实能快速做出能跑的东西。但很快遇到了和上面一样的问题--爽快开始、混乱中段、放弃收尾。

我才意识到：**小项目也需要架构。** 我缺的不是编码能力（AI 补上了），而是架构思维 🧠。

既然小项目也要学架构，为什么不直接学一个成熟软件的架构呢？于是我又转回了 opencode--毕竟它是一个经过真实用户验证的生产级软件，它的架构决策值得每一行去理解。只是这次换了个学法：不是硬读源码，而是从简化版开始，一步步演进到和它一致。

### 🌀 看视频的死胡同

我试过"先补基础再看源码"。在 B 站找了一堆技术栈教程：

> HTML、CSS、JavaScript、TypeScript、React、Vue、Bun、SQLite、Drizzle、Effect-TS……

看了两周，感觉越看越晕。**前后端的技术栈太庞大了。** 🌊 每个技术都有几十小时的视频，学完一个发现后面还排着十个。而且视频教的是"这个技术怎么用"，不是"为什么要用这个技术"--你看了一堆工具的说明书 📚，但还是不知道一座大楼是怎么设计出来的 🏗️。

### ✨ 转变：AI + Agent 让"边造边学"成为可能

后来想通了：**不要先学技术再造东西，而是边造东西边学技术。** 🔑

而之所以这条路走得通，是因为复刻 opencode 这个目标天然适合让 AI + agent 来拆解 🤖。opencode 是一个成熟的软件，它的源码就是最好的教材。agent 可以把庞大的知识点打碎--用到 Effect 的 Service，就让它讲 Service；用到 Drizzle 的表设计，就让它讲表设计。**用到什么学什么，不需要提前啃完一整本教程。**

更关键的是，AI 出的教程你可以反复 battle ⚔️。看不懂？让它换个说法。还是不懂？让它用 Python 类比。太抽象？让它给个生活中的例子。一遍遍打磨到你真正理解为止。**这是传统教育完全做不到的**--没有哪个老师能陪你反复改一份讲义，直到它刚好适合你的认知水平。

**先感受到痛点，再学解决方案，而且有 AI 随时把每个知识点讲到你能懂--理解才深刻。** 💡

这就是这个项目的由来：从最简化的能跑版本起步，逐步演进到 opencode 的完整 1:1 复刻。每一阶段先实现一个"够用但简化"的版本让东西跑起来，理解它解决什么问题，再逐步补全到与 opencode 源码一致。

## 📦 这个项目是什么

**一个教学项目**：从零开始、一步一步重新实现 opencode。

| | |
|---|---|
| 📌 参考版本 | opencode `v1.17.13`（commit `10c894bde`，分支 `dev`） |
| 📂 源码参考 | `./opencode/` 目录（只读，不修改） |
| 🎯 最终目标 | opencode 的完整 1:1 复刻 |

opencode 的核心是一个 **agent loop**：

```
接收用户指令 -> 组装上下文 -> 调用 LLM -> 执行工具调用 -> 把结果喂回 LLM -> 循环直到完成
```

看起来简单，但生产级的实现涉及大量架构决策：怎么管理依赖、怎么持久化状态、怎么处理流式、怎么解耦 UI 和业务逻辑、怎么支持多 LLM 厂商……

这个项目不是教程式的"跟着我敲"，而是 **"一起想清楚为什么"** 🧐。每段代码都应该能回答"为什么这样写"。

## 🎓 你能从中学到什么

这个项目有完整的配套教程 📚，不只是代码，还讲解整个项目是如何一步步搭建出来的。你可以学到：

- **🔧 opencode 的使用**：不只是当用户用，而是理解它内部的每一个设计决策--为什么用 Effect-TS、为什么用事件溯源、为什么拆成 31 个 package
- **🤖 如何让 AI agent 做出适合自己的教程**：这个项目的教程本身就是用 AI agent 辅助生成的。你可以学到怎么把庞大的知识点拆碎、怎么和 AI 反复 battle 打磨讲义、怎么让 AI 用你能理解的方式解释陌生概念
- **🧑‍💻 作者的学习路线和心得感悟**：每个阶段都有"工程思维总结"，记录作者踩过的坑、想通的瞬间、为什么这样设计而不是那样。当然，每个人的情况不同，这个课程安排不一定完全适合你。但你完全可以按照同样的思路，用 AI agent 为自己构建一份适合自己节奏和背景的教程

> 💡 这个项目的真正价值不是"教会你 opencode 怎么写"，而是"展示一种 AI 时代的学习方法"--边造边学，让 AI 把每个知识点讲到你能懂。

## 🗺️ 学习路线

### 第一阶段：简化版能跑（阶段 0-9 ✅ 已完成）

用最少的抽象把 agent loop 跑通，理解核心机制：

| 阶段 | 内容 | |
|------|------|--|
| 0 | 环境与基础（TypeScript + Bun 起步） | ✅ |
| 1 | 最小 Agent（一次 LLM 调用） | ✅ |
| 2 | 流式输出 | ✅ |
| 3 | 工具循环（Agent 的核心） | ✅ |
| 4 | 工具集（read/write/edit/bash/grep/glob） | ✅ |
| 5 | Session 持久化（SQLite + Drizzle） | ✅ |
| 6 | Provider 抽象 | ✅ |
| 7 | System Context & AGENTS.md | ✅ |
| 8 | CLI 入口（yargs） | ✅ |
| 9 | TUI 终端界面（opentui/solid） | ✅ |

### 第二阶段：演进到 1:1 复刻（阶段 10-21 🔧 进行中）

每个阶段因为一个具体痛点引入 opencode 的对应抽象：

| 阶段 | 解决的痛点 | 引入的抽象 |
|------|-----------|-----------|
| 10 | 依赖到处传 | Effect Service/Layer/Stream/Schema 🔨 |
| 11 | 类型重复、边界模糊 | Schema 契约层 + Bun workspaces |
| 12 | 领域逻辑散乱 | Core 领域服务化 |
| 13 | 无法 revert/恢复/压缩 | Session 事件溯源 |
| 14 | 加 provider 要复制粘贴 | LLM Route 四轴模型 |
| 15 | TUI 与 agent 耦合 | Server + Protocol + Client |
| 16 | 工具能乱改无确认 | Permission 系统 |
| 17 | 单 agent 干所有事 | Agent 定义 + Subagent |
| 18 | 工具不够用 | 完整工具集 |
| 19 | 工具扩展要改源码 | MCP 支持 |
| 20 | 长对话爆上下文 | Compaction + revert + plugin + LSP |
| 21 | 只有 TUI | Web UI + Desktop |

> 核心原则：**动机驱动 + 渐进演进** 🎯。不是"opencode 有什么就照搬什么"，而是"哪里痛了才引入什么"。

## 🛠️ 技术栈

| 技术 | 作用 | 对应 Python 概念 |
|------|------|------------------|
| **Bun** | JS 运行时 + 包管理器 | Python 解释器 + pip/uv |
| **TypeScript** | 编程语言 | 带类型标注的 Python |
| **Effect-TS** | 函数式框架（Service/Layer/Stream/Schema） | 带依赖注入的 async + Result 类型 |
| **Drizzle ORM** | SQLite ORM | SQLAlchemy |
| **yargs** | CLI 参数解析 | argparse / click |
| **opentui/solid** | 终端 UI 框架 | 无直接对应；类似 React 但渲染在终端 |
